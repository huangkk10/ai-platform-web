# 📖 測試執行頁面 - 使用指南

## 🎯 快速開始

### 1. 進入測試執行頁面
```
側邊欄 → 測試執行
或直接訪問：http://localhost/benchmark/test-execution
```

### 2. 執行測試（4 步驟）

#### 步驟 1：選擇演算法版本
- 點擊「選擇版本」下拉選單
- 會自動選擇「基準版本」（藍色標籤）
- 或手動選擇其他版本

#### 步驟 2：輸入測試名稱
- 在「測試名稱」欄位輸入描述性名稱
- 範例：`初始測試 v1.0`、`優化後測試`
- 最多 200 字元

#### 步驟 3：選擇測試類型
**選項 A：開始完整測試** 🚀
- 執行所有可用的測試案例
- 預估時間：根據題數自動計算
- 適合：完整評估演算法效能

**選項 B：快速測試 (5題)** ⚡
- 隨機選擇 5 題測試
- 預估時間：約 10-15 秒
- 適合：快速驗證改動效果

#### 步驟 4：等待測試完成
- 進度條會每 2 秒自動更新
- 顯示已完成題數 / 總題數
- 可點擊「停止測試」中斷執行

### 3. 查看結果
- 測試完成後會顯示結果摘要：
  - ✅ 總分
  - ✅ 完成題數
  - ✅ 執行時間
- 3 秒後自動跳轉到 Dashboard
- 可在 Dashboard 查看歷史記錄

## 🎯 使用案例

### 案例 1：新演算法初次測試
```
1. 選擇版本：新建立的演算法版本
2. 測試名稱：「演算法 v2.0 初次測試」
3. 測試類型：快速測試 (5題)
4. 目的：快速驗證基本功能
```

### 案例 2：完整效能評估
```
1. 選擇版本：Baseline Version（基準版本）
2. 測試名稱：「基準版本完整評估 - 2025-11-22」
3. 測試類型：開始完整測試
4. 目的：獲得完整效能數據
```

### 案例 3：參數調整驗證
```
1. 選擇版本：調整參數後的版本
2. 測試名稱：「權重調整測試 - 增加語義搜尋權重」
3. 測試類型：快速測試 (5題)
4. 目的：快速驗證調整效果
```

## ⚠️ 注意事項

### 測試執行期間
- ❌ **不要關閉瀏覽器**：會中斷測試
- ❌ **不要重新整理頁面**：進度會丟失
- ✅ **可以停止測試**：點擊「停止測試」按鈕

### 測試名稱建議
- ✅ 包含日期：方便日後查詢
- ✅ 描述性：說明測試目的
- ✅ 簡潔明瞭：避免過長

### 版本選擇
- 🔵 **基準版本**：用於效能比較的參考版本
- ⚪ **其他版本**：測試中或實驗性的版本

## 🔧 故障排除

### 問題 1：版本下拉選單顯示「無資料」
**解決方法**：
1. 點擊右上角「重新整理」按鈕
2. 確認您已登入系統
3. 檢查是否有建立演算法版本

### 問題 2：測試啟動失敗
**可能原因**：
- 未選擇版本
- 未輸入測試名稱
- 權限不足

**解決方法**：
1. 確認所有必填欄位已填寫
2. 確認您已登入
3. 查看錯誤訊息提示

### 問題 3：測試一直執行不完成
**解決方法**：
1. 點擊「停止測試」按鈕
2. 檢查 Django 後端日誌
3. 重新啟動測試
